# ESP32 Template - Copilot Instructions

## Available Commands

### 1. Create New ESP32 Project
**Syntax:** `new-project <ProjectName> <BoardType>`

**Example:**
```
new-project my-device C3
new-project sensor-hub ESP32
new-project iot-watch S2
```

**Supported Board Types:**
- `ESP32` — Classic, Dual-Core, WiFi + BLE + Classic Bluetooth
- `S2` — Single-Core, WiFi only (no BLE), Ultra-low-power
- `S3` — Dual-Core, WiFi + BLE 5.0, AI Accelerator
- `C3` — Single-Core, WiFi + BLE 5.0, Compact RISC-V
- `C6` — Single-Core, WiFi 6 + BLE 5.3 + 802.15.4, Best power efficiency

**What this does:**
1. Klones template → `C:\Users\win4g\Downloads\GitHub\VS-Projekte\<ProjectName>/`
2. Initializes new Git repo (clean, not cloned)
3. Creates `PROJECT.md` with board-specific defaults
4. Creates `CHANGES.md` for feedback to template
5. Adjusts `platformio.ini` for selected board
6. First commit: "Initial commit: ESP32 Template (<BoardType>)"

**Example Output:**
```
✓ Project created: C:\Users\win4g\Downloads\GitHub\VS-Projekte\my-device/
✓ Board: ESP32-C3 (Single-Core, WiFi+BLE)
✓ Git initialized
✓ Ready to code
```

## When to Use Each Board

| Board | Best For | Power | Notes |
|-------|----------|-------|-------|
| **ESP32** | General IoT, Smart Home, Audio | 50-100 mA | Dual-Core, most mature |
| **S2** | WiFi-only sensors, ultra-low-power | 40-80 mA | No Bluetooth |
| **S3** | AI/ML, Computer Vision, Audio | 60-100 mA | Dual-Core, Neural Engine |
| **C3** | BLE wearables, compact projects | 30-40 mA | Single-Core, RISC-V, smallest |
| **C6** | Battery devices, WiFi6, Thread/Matter | 20-30 mA | Best efficiency, modern IoT |

## Project Structure

Nach `new-project <name> <board>` entsteht:

```
<ProjectName>/
├── IMPLEMENTATION_PLAN.md      # Wie Projekte entstehen sollen
├── plan.md                      # ESP32 Template Roadmap (kopiert)
├── PROJECT.md                   # Board-spezifische Details (editierbar)
├── CHANGES.md                   # Feedback zum Template (editierbar)
├── README.md                    # Projekt-Anleitung
├── platformio.ini               # PlatformIO Config (angepasst für Board)
├── src/main.cpp                 # Arduino Grundgerüst
├── include/config.h             # Pin-Platzhalter
├── lib/                         # Custom Libraries
├── test/                        # Unit Tests
├── .git/                        # Fresh Git Repo
└── .agent.md                    # Agent Instructions (referenziert plan.md)
```

## Workflow

1. **Create Project**
   ```
   new-project my-first-project C3
   ```

2. **Setup Hardware** (PROJECT.md)
   - Document GPIO Pins
   - WiFi/BLE Configuration
   - Power Profile

3. **Develop** (src/main.cpp, include/config.h)
   - Write your code
   - Follow plan.md guidelines

4. **Document Changes** (CHANGES.md)
   - If something should improve the template:
     `"Add WiFi DefaultSSID config to template"`
   - We discuss → improve template for next projects

5. **Commit & Push**
   - `git add .`
   - `git commit -m "Feature: WiFi connection"`
   - `git push origin main`

## Troubleshooting

**Script not found?**
- Ensure you're in the template directory
- Check `new-project.ps1` exists
- Run: `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

**Board not supported?**
- Run: `show boards` to see available options
- All ESP32 variants with FreeRTOS support are included

**Project creation failed?**
- Check disk space
- Verify path: `C:\Users\win4g\Downloads\GitHub\VS-Projekte\` is writable
- Check if ProjectName already exists

## Notes

- `plan.md` in each project is a **copy** of the template — it won't auto-update
- `PROJECT.md` and `CHANGES.md` are **editable** and project-specific
- Template improvements are discussed manually before merging
- All boards support FreeRTOS (configuration differs)
