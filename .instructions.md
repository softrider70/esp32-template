# Copilot Custom Instructions for ESP32 Template

## Available Skills (/commands)

All commands are available as `/skillname` in the chat.

### Phase 1-3: Core Project Management

#### `/new-project` - Create new ESP32 project
- Interactive project generation
- Board selection (5 variants)
- Input validation
- Git initialization
- **Usage:** `/new-project`

#### `/build-project` - Compile firmware
- Build current ESP32 project
- Auto-detect board target
- Cross-board support (ESP32, S2, S3, C3, C6)
- **Usage:** `/build-project`

#### `/upload` - Smart upload router
- Intelligent routing (first-time vs. update)
- Auto-detects upload type
- Safe partition handling
- **Usage:** `/upload`

#### `/upload-firmware` - Fast app update
- App-only update (~3 seconds)
- Bootloader unchanged
- Perfect for iteration
- **Usage:** `/upload-firmware`

#### `/initial-upload` - Full first-time setup
- Bootloader + partition table + app (~20 seconds)
- One-time setup only
- Full device provisioning
- **Usage:** `/initial-upload`

#### `/commit` - Smart git operations
- Auto-stage changes
- Generate commit messages
- Push to git
- **Usage:** `/commit`

### Phase 4: Advanced Feature Skills

#### `/add-ota` - Over-the-air firmware updates
- HTTP/HTTPS download integration
- Partition management (ab-scheme)
- Firmware verification (SHA256)
- Rollback mechanism
- **Files Generated:** ota_handler.h/c, ota_config.h
- **Usage:** `/add-ota`

#### `/add-webui` - Web UI generator
- Responsive HTML/CSS/JS templates
- REST API client
- WebSocket support (optional)
- Dark/Light theme toggle
- Mobile-friendly
- **Files Generated:** webui/{html,js,css,server}/
- **Usage:** `/add-webui`

#### `/add-library` - Component registry
- ESP-IDF registry integration
- Dependency management
- Version control
- Conflict detection
- **Usage:** `/add-library COMPONENT_NAME`
- **Example:** `/add-library mqtt`

#### `/add-security` - Production security
- Secure Boot V2 (signature verification)
- NVS partition encryption
- TLS/SSL certificate management
- Hardware security (eFuse)
- **WARNING:** Secure Boot is IRREVERSIBLE!
- **Files Generated:** security_config.h, keys/, SECURITY_GUIDE.md
- **Usage:** `/add-security`

#### `/setup-ci` - GitHub Actions CI/CD
- Multi-board parallel builds (5 variants)
- Automated testing and analysis
- Release automation
- Status badges
- **Files Generated:** .github/workflows/
- **Usage:** `/setup-ci`

#### `/add-profiling` - Performance analysis
- Real-time heap monitoring
- Stack overflow detection
- Performance timing macros
- Memory fragmentation analysis
- Live UART dashboard
- **Files Generated:** heap_profiler.h/c, stack_monitor.h/c, perf_timer.h
- **Usage:** `/add-profiling`

## Quick Reference

### Project Creation Workflow
```
1. /new-project          (create new project)
2. cd my-project
3. /build-project        (compile)
4. /upload-firmware      (flash app)
5. /commit               (save changes)
```

### Adding Features
```
/add-webui               (responsive UI)
/add-ota                 (firmware updates)
/add-security            (Secure Boot, NVS)
/add-library mqtt        (components)
/setup-ci                (CI/CD pipeline)
/add-profiling           (memory analysis)
```

### Development Cycle
```
1. Edit code
2. /build-project
3. /upload-firmware      (fast iteration)
4. Test
5. /commit
```

## Supported Boards

All skills support 5 ESP32 variants:
- **ESP32** (dual-core, 520KB SRAM + PSRAM)
- **ESP32-S2** (single-core, 320KB SRAM)
- **ESP32-S3** (dual-core, 512KB SRAM + PSRAM)
- **ESP32-C3** (single-core RISC-V, 400KB SRAM)
- **ESP32-C6** (dual-core RISC-V, 512KB SRAM + PSRAM)
